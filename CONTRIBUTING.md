# Contributing to AURA

First off, thank you for considering contributing to AURA! It's people like you that make open-source ecosystems incredible.

## How Can I Contribute?

### 🐛 Reporting Bugs
This section guides you through submitting a bug report for AURA.
- **Check existing issues:** Ensure the bug hasn't already been reported.
- **Use a clear title:** Describe the issue succinctly.
- **Include steps to reproduce:** How can we trigger the bug?
- **Provide environment details:** OS, CPU model (for OpenVINO), Flutter version, Bun version.

### 💡 Suggesting Enhancements
Got an idea for a new Local Agent Tool, a UI tweak, or a core architectural improvement?
- **Detail your proposal:** What is the feature and why is it needed?
- **Provide mockups or diagrams:** If it's a visual or architectural change, diagrams help immensely.

### 💻 Pull Requests
1. **Fork the repo** and create your branch from `main`.
2. **If you've added code** that should be tested, add tests.
3. **Ensure the test suite passes.**
4. **Make sure your code lints.** 
5. **Issue that PR!**

## Development Environment Setup

Please refer to the `README.md` for standard setup. If you are developing locally:
1. Ensure the Python `venv` is active when modifying the Local Agentic Brain.
2. For Flutter UI changes, we recommend testing on `chrome` first with the `--web-renderer canvaskit` flag.
3. If altering the Bun Gateway, ensure you restart the server to catch routing changes.

## Code of Conduct
Please note that this project is released with a Contributor Code of Conduct. By participating in this project you agree to abide by its terms. Be respectful and constructive!

Thank you for helping us build the future of localized Agentic AI! 🚀
